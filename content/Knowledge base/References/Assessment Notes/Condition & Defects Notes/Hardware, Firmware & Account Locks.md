Below is a list of possible hardware locks found on different devices
## Bios Lock
```
The device is BIOS locked, preventing configuration changes such as enabling external boot for testing. As a result, full functionality testing could not be performed.
```
## Activation Lock
```
The device was not properly signed out of and remains associated with an Apple ID, resulting in an active Activation Lock. This security feature prevents the device from being set up or used by another user without the credentials of the original account holder. Until the Apple ID is removed and the Activation Lock is cleared, the device cannot be fully reset or configured for new use. Verification of account removal is required before the unit can be considered unlocked and operational.
```
## Find My
```
the unit was not properly signed out, and Find My remains active. As a result, the unit is considered still logged into an Apple ID and is categorized as Locked. The device cannot be fully reset or reassigned until the associated account is removed and verified.
```
## Factory Reset Protection
```
The Pixel Watch was not properly signed out of before being reset, and it remains associated with a Google account. As a result, the device is locked with Factory Reset Protection (FRP), preventing it from being set up or used by another user. This security measure ensures that only the original account holder can reactivate the device. The last-used PIN/pattern to reactivate the device is required before the watch can be configured and made fully functional again.
```
## Firmware Lock
```
The device’s firmware is locked, preventing configuration, operating system installation, and normal booting. This restriction renders the unit nonfunctional until the firmware lock is removed or reset.
```
